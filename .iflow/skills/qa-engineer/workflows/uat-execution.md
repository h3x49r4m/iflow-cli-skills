# User Acceptance Testing (UAT) Workflow

## Objective
Execute UAT to ensure product meets stakeholder expectations.

## Steps

1. **Prepare for UAT**
   - Read `project-spec.md` for acceptance criteria
   - Review `quality-report.md` for known issues
   - Prepare UAT environment
   - Create UAT test scripts

2. **Conduct UAT with Stakeholders**
   - Schedule UAT sessions with stakeholders
   - Provide clear instructions
   - Monitor stakeholder testing
   - Collect feedback

3. **Validate Acceptance Criteria**
   - Verify each acceptance criteria is met
   - Document pass/fail status
   - Capture stakeholder feedback
   - Identify issues

4. **Document UAT Results**
   - Update `quality-report.md` with UAT results
   - Include stakeholder feedback
   - Document issues and fixes
   - Provide sign-off status

## Output
- Updated `quality-report.md` with UAT results
- Updated `pipeline-status.md` with completion status
