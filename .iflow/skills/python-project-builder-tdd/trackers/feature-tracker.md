## Using This Tracker

1. **Add Feature**: When a feature is planned, add it to this tracker
2. **Update Status**: As feature progresses, update status and phase
3. **Check Progress Items**: Mark items complete as they're done
4. **Track Acceptance Criteria**: Update as criteria are met and tested
5. **Update Coverage**: Record test coverage percentages
6. **Log Time**: Track estimated vs actual time
7. **Document Issues**: List any bugs or issues found
8. **Note Dependencies**: Track feature dependencies

## Notes

- Update this tracker regularly
- Use feature IDs (F001, F002, etc.) for consistency
- Mark current phase clearly
- Keep acceptance criteria up to date
- Note any blockers or delays
- Review weekly to identify risks
